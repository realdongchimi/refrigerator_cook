# 냉장고를 부탁해 PRD (Product Requirements Document)

## 1. 개요

**냉장고를 부탁해**는 Gemini AI를 활용한 사진 분석(식재료) 기반 자동 요리 추천 및 레시피를 출력하는 iOS 앱

### 핵심 가치 제안
- **문제** : 무슨 음식을 요리해야할지 모르는 '결정 장애' 발생, 외식 및 배달로 건강과 경제적인 문제 발생
- **해결책** : AI가 사진촬영으로 식재료를 분석 후 메뉴 추천 및 레시피 작성
- **차별점** : 무료 Gemini API로 비용 부담 없음, 사진 촬영으로 식재료 분석

---

## 2. 타겟 사용자
### 주요 사용자
- 냉장고에 식재료는 있지만 무슨 요리를 할지 **결정하지 못하는 현대인**

### 페인포인트
- 요리할 시간이 부족하거나, 매번 새로운 메뉴를 고민하기 어려운 사용자.
- 식비 절약과 건강한 집밥을 선호하지만, 재료 활용에 어려움을 겪는 사용자.

---

## 3. 핵심 기능 (MVP)
### P0 - 필수 기능
- AI 사진 인식 : 사용자가 냉장고 식재료를 촬영하면 AI가 자동으로 재료를 구분하여 목록화하는 기능.
- AI 자동 추천 : 인식된 식재료 목록을 기반으로 AI (Gemini)가 오늘 요리할 메뉴를 **자동으로 추천**해주는 기능.
- AI 레시피 제공 : 추천된 메뉴를 선택하면, AI (Gemini)가 해당 요리의 상세한 레시피(재료, 조리 단계, 시간 등)를 제공하는 기능.

### P2 - 향후 기능
- 추천 기록 관리 : 사용자가 추천받은 요리 목록과 실제로 선택한 요리를 저장하여 기록을 관리하는 기능.
- 소셜 : 다른 유저가 올린 레시피 확인

---

## 4. 기술 스택

* **개발 환경:**
    * **OS:** iOS 16.0+
    * **아키텍처:** MVVM (Model-View-ViewModel) + SwiftData (데이터 저장 및 관리)
* **AI 모델:** Google Gemini API
* **주요 라이브러리:**
    * **시각화:** Swift Charts
    * **애니메이션:** Lottie
    * **UI:** SwiftUI Components (Custom UI)

---

### 5. Gemini API 통합 전략

| 기능 | Gemini 역할 (통합 전략) | 담당 |
| :--- | :--- | :--- |
| **코드 전반의 틀** | Swift 코드 작성 및 전반적인 개발 프레임워크 지원. | AI 담당 |
| **AI 자동 추천** | **입력:** 앱에서 추출한 식재료 목록. **출력:** 식재료를 활용한 요리 메뉴 추천 (3~5가지). | AI 담당 |
| **AI 레시피 제공** | 추천된 요리에 대한 **상세 레시피** (재료, 조리 단계, 소요 시간)를 생성하여 출력. | AI 담당 |
| **AI 코드 검토** | AI 생성 코드의 안정성, 효율성, 그리고 Swift 언어 및 iOS 환경 적합성 검토. | 내 담당 |

---

### 6. UI/UX 원칙

* **단순성 (Simplicity):** 사진 촬영부터 추천까지의 과정을 **최소한의 탭**으로 완료하도록 설계.
* **시각적 명확성:** 인식된 식재료 목록과 추천 요리 결과는 **Swift Charts**를 활용하여 명확하고 흥미롭게 시각화.
* **친근함:** '결정 장애'를 해소하는 긍정적인 경험을 주기 위해 **Lottie** 애니메이션을 활용하여 부드럽고 재미있는 사용자 인터랙션 제공.
* **피드백:** 사진 인식 오류 시 즉각적인 수정 피드백 및 레시피 조리 단계별 명확한 안내 제공.

### 7. 성공 지표 (KPI: Key Performance Indicators)

| 지표 | 정의 | 목표 |
| :--- | :--- | :--- |
| **사용자당 월평균 사용 횟수** | 월간 앱을 실행하여 요리 추천을 받은 횟수 | **5회 이상** (주 1회 이상 사용) |
| **추천 수용률** | AI가 추천한 요리 중 사용자가 레시피를 조회하거나 저장한 비율 | **50% 이상** |
| **사진 인식 정확도** | AI 사진 인식 기능이 실제 식재료를 정확하게 파악하는 비율 | **90% 이상** |
| **이탈률** | 추천 기능을 사용하려다 도중에 앱을 종료하는 사용자 비율 | **10% 미만** |

### 8. 사용자 스토리 (User Stories)

| ID | 사용자 스토리 | 중요도 |
| :--- | :--- | :--- |
| US-01 | **결정 장애 사용자**로서, 냉장고 사진을 찍으면 **자동으로 요리 메뉴를 추천**받아 고민 없이 저녁 메뉴를 정하고 싶다. | High |
| US-02 | **집밥 초보자**로서, 추천받은 요리를 클릭하면 **단계별 상세 레시피**를 확인하여 쉽게 요리할 수 있도록 하고 싶다. | High |
| US-03 | **식재료 절약가**로서, AI가 인식한 식재료 목록을 **직접 수정**하여 정확한 재료를 기반으로 추천받고 싶다. | Medium |
| US-04 | **건강을 챙기는 사람**으로서, 내가 해 먹었던 요리 목록을 **기록**으로 남겨 식단을 관리하고 싶다. | Medium |

### 9. 비기능 요구사항 (Non-Functional Requirements)

* **성능:** AI 사진 인식 및 요리 추천 응답 시간은 **3초 이내**로 제한.
* **보안:** 사용자 촬영 사진은 서버에 저장하지 않으며, API 통신은 **HTTPS**를 통해 암호화.
* **사용성:** iOS 접근성 기능을 지원하여 시각/청각 장애 사용자도 사용할 수 있도록 설계.
* **확장성:** 향후 요리 난이도, 조리 시간, 선호 재료 등 **개인화 필터**를 추가할 수 있도록 모듈화된 아키텍처 (MVVM)로 설계.

### 10. 측정 가능한 목표 (Measurable Goals)

* **출시일:** 2026년 Q1 (개발 일정에 따라 변동 가능)
* **사용자 수:** 출시 3개월 이내 **1,000명 이상의 활성 사용자** 확보.
* **평점:** App Store 사용자 평점 **4.5점 이상** 달성.
* **버그율:** 베타 테스트 기간 동안 치명적인 버그 발생률 **0.1% 미만** 유지.
